Review Strategy

윤식단 리뷰 UI는 네 가지 질문에 답해야 합니다

기존처럼 평점과 최신 리뷰만 나열하면 고객은 “내 상황에도 맞을까?”를 다시 스크롤해서 찾아야 합니다. 위젯은 그 탐색 비용을 먼저 줄이는 역할이어야 합니다.

Barrier 01

양과 포만감

다이어트 도시락은 적을 것 같다는 의심을 “3시까지 버팀”, “밥이 있어 든든함” 리뷰로 바로 해소합니다.

Barrier 02

맛과 질림

맛있다는 말보다 “돌고 돌아 윤식단”, “소스/구성 조합”처럼 반복 구매 이유를 노출합니다.

Barrier 03

직장 점심 루틴

사무실, 점심, 야근, 냉동실 쟁여두기처럼 실제 사용 장면을 기준으로 리뷰를 필터링합니다.

Barrier 04

냉동 배송 신뢰

냉동식품 특성상 배송/보관 불안을 별도 키워드로 분리해 FAQ와 연결합니다.

Implementation Map

실제 세팅은 위젯을 많이 넣는 게 아니라 역할을 나눠야 합니다

라라스윗/알파리뷰식 접근을 그대로 복제하지 않고, 윤식단 구매 장벽에 맞춰 위젯을 최소한으로 배치합니다.

1. 상품 상단 브리프평점, 리뷰수, 핵심 키워드, 대표 리뷰 2개만 노출해 구매 옵션 진입 전에 불안을 줄입니다.
2. 상세 중간 포토 프루프긴 상세 설명 사이에 실제 식사 사진과 상황 키워드를 넣어 스크롤 이탈을 줄입니다.
3. 구매평 탭 리뷰 허브최신순이 아니라 상황별 필터와 키워드 통계로 리뷰 2만 개를 탐색 가능한 자산으로 바꿉니다.
4. 모바일 하단 플로팅장바구니/구매 버튼 근처에는 짧은 사회적 증거만 띄우고, 상세한 리뷰는 탭으로 이동시킵니다.

이번 시안의 기준

단순 리뷰 위젯이 아니라, 라라스윗/알파리뷰식 리뷰 운영을 윤식단 상품 상세에 맞게 재구성했습니다.

라라스윗에서 본 패턴

  • 상품 구매 영역은 가격/옵션/구매에 집중
  • 리뷰는 별도 REVIEW 영역으로 모아 신뢰 자산화
  • 브랜드 톤은 강하지만 상품 정보 구조는 단순

알파리뷰에서 쓸 기능

  • 스마트스토어 리뷰 연동
  • 베스트 리뷰 지정 위젯
  • 키워드 리뷰 위젯
  • 포토 리뷰 / 플로팅 리뷰
  • 위젯 순서와 노출 개수 설정

윤식단에 맞춘 운영 룰

  • 상단 대표 리뷰는 “맛있어요”보다 “점심/포만감/재구매” 우선
  • 포토 리뷰는 제품 실물보다 식사 맥락이 보이는 컷 우선
  • 부정/중립 리뷰도 배송/보관 FAQ와 연결해 투명성 확보
  • 리뷰 작성 유도 항목은 양, 맛, 조리, 재구매 의향으로 설계
